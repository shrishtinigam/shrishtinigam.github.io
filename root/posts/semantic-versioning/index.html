Applying semantic versioning the right way can prevent dependency issues when working with multiple libraries and meta-packages.

Definition

Semantic Versioning (SemVer) is a versioning system used in software development to track changes in a structured way and reduce dependency issues.

Commit Style

This part of the article is something I care about deeply personally, but not truly a part of SemVer. When applied to commit messages, it helps teams maintain consistent, meaningful, and easy-to-understand commit histories.

The core idea behind this style is to categorize commits based on their impact (features, fixes, documentation updates, etc.), which makes it easier to generate changelogs, automate releases, and maintain a clean Git history.

Type Purpose
feat: Adds a new feature
fix: Fixes a bug
chore: Maintenance tasks
build: Build system or dependencies
docs: Documentation updates
perf: Performance improvements
refactor: Code refactoring
style: Code formatting
test: Adding or modifying tests
ci: CI/CD changes
revert: Reverting a commit
BREAKING CHANGE: Major breaking changes
