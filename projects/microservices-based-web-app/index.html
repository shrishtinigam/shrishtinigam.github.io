Overview

Built a scalable E-Commerce Web Application using a microservices and event-driven architecture.

Highlights

  • Backend implemented with Node.js and TypeScript.
  • Minimalist front-end using React and Next.js with server-side rendering.
  • Deployed using Docker and Kubernetes for scalability.
  • Orders managed efficiently using Redis and Bull.js for queueing.

Repository

ticket-house - GitHub Link

Ticket House

An E-Commerce Web Application with Microservices and Event-Based Architecture.

Technologies Used

  • Backend / Microservices
    Node.js, Express, TypeScript

  • Frontend / Rendering
    React, Next.js (server-side rendering)

  • Communication / Event Architecture
    NATS as event bus

  • Infrastructure / Deployment / Jobs
    Docker, Kubernetes, Bull.js + Redis for job queues

  • Payments
    StripeJS

Setup Instructions

To get the project running locally:

  1. Clone the repository (ideally under WSL2 on Windows)
  2. Make sure Docker is running and Kubernetes is enabled
  3. Create a Kubernetes secret for JWT:
    bash kubectl create secret generic jwt-secret --from-literal=JWT_KEY=random_secret

Development Setup

Run the development environment:

skaffold dev

Architecture

  • Microservices architecture: independent services such as auth, tickets, orders, payments
  • Event-based communication through NATS for decoupling and asynchronous flows
  • Job queue with Bull.js + Redis for background tasks or delayed operations
  • Deployment and orchestration using Docker + Kubernetes, with local development via skaffold

Usage / Contribution

  1. Ensure you have Docker and Kubernetes ready.
  2. Set up the JWT secret.
  3. Run skaffold dev to spin up all services + frontend.

If contributing, you might explore or extend: - Adding more services (e.g., shipping, user profiles)
- Improving error handling / observability
- Adding tests for event flows
- Enhancing performance, scaling, or security
